The bug was caused by the `MultiplePiePlot` constructor directly assigning the `dataset` variable instead of using the `setDataset` method, which bypassed the logic for registering the plot as a listener to the dataset. As a result, changes to the dataset would not trigger updates in the plot, leading to inconsistent or outdated visualizations. Evidence for this issue is provided by the failing `testConstructor` unit test, which checks whether the plot is correctly registered as a listener to the dataset after construction. The bug was resolved by replacing the direct assignment with a call to `setDataset`, ensuring the plot is properly registered and all associated logic is executed.
