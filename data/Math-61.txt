The bug was caused by the incorrect exception being thrown when the mean parameter (`p`) of the `PoissonDistributionImpl` constructor was non-positive; instead of throwing a `NotStrictlyPositiveException`, the code threw a generic `MathRuntimeException` with a localized message. This led to the application failing to properly signal the specific error condition, causing confusion for developers or users expecting the more precise `NotStrictlyPositiveException` to handle invalid input. Evidence for this issue is found in the `testMean` unit test, which explicitly expects a `NotStrictlyPositiveException` and fails if any other exception type is thrown. The bug was resolved by replacing the call to `MathRuntimeException.createIllegalArgumentException` with a direct instantiation of `NotStrictlyPositiveException`, ensuring the correct exception type is thrown and aligning the implementation with the expected behavior.
