The bug in the `BrentSolver` class was caused by an incorrect condition (`sign >= 0`) used to verify bracketing, which improperly included cases where `sign == 0`, leading to unnecessary exceptions when the function values at the endpoints were both zero. This impacted the solver by failing to return a valid root at an endpoint, instead throwing an `IllegalArgumentException` and breaking functionality for certain inputs. Evidence for this issue comes from the failing unit test `testRootEndpoints`, which demonstrates the solver's inability to handle intervals where one endpoint is a root, such as `Math.PI`. The bug was resolved by changing the condition to `sign > 0` and adding logic to handle cases where the function values at the endpoints are close to zero or exactly zero, ensuring the solver correctly identifies and returns the root at the endpoint.
