This allowed invalid `elitismRate` values (e.g., less than 0 or greater than 1) to be set during object construction, potentially leading to runtime errors or incorrect behavior in genetic algorithm operations. Evidence for this issue comes from unit tests such as `testConstructorTooHigh` and `testConstructorTooLow`, which failed to trigger validation errors when invalid values were used. The bug was resolved by replacing the direct assignment of `elitismRate` in the constructors with calls to the `setElitismRate` method, ensuring consistent validation during object creation.
