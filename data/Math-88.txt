The bug was caused by the incorrect handling of duplicate basic rows in the simplex tableau, leading to erroneous calculations of the `coefficients` array when multiple variables shared the same basic row. This resulted in invalid optimization solutions, as non-zero values were assigned to variables that should have been zero, causing test cases like `testMath272` to fail. Evidence supporting this analysis comes from the failing `testMath272`, which showed incorrect decision variable values due to shared basic rows not being properly zeroed out. The bug was resolved by introducing a `Set<Integer>` called `basicRows` to track processed basic rows, ensuring that coefficients for variables sharing the same basic row are set to zero, thereby producing correct optimization results.
